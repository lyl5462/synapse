# 会议室重启、停止与重处理方案

本文档定义研发会议室在 **后端服务重启**、**用户终止节点运行**、**重新处理** 三类场景下的状态语义与落点。与 [多智能体研发会议室实现方案](./多智能体研发会议室实现方案.md) 互补，聚焦 `room_state.status` 与前端会议室弹窗。

---

## 1. 新增状态 `stopped`

| 状态 | 语义 |
|------|------|
| `processing` | 后台应自动执行当前节点（本进程内可能有 `run_in_progress`） |
| `human_intervention` | 待人机表单/结果确认，**不**自动跑 |
| `failed` | 流程失败（校验、用户 abort 等） |
| `completed` | 会议/SOP 已走完 |
| **`stopped`** | 人为终止或服务重启后冻结；**保留**磁盘进度，**不**自动续跑 |

辅助字段（写入 `room_state.json`）：

- `stopped_at`：ISO 时间
- `stopped_reason`：`server_restart` | `user_stop`

协作流事件：`room_stopped`（`log_type: warning`）。

---

## 2. 服务重启：批量标 `stopped`

### 2.1 触发

`synapse serve` 进程 **startup** 时执行一次（`mark_active_rooms_stopped_on_server_restart`）。

### 2.2 扫描范围

`{SYNAPSE_HOME}/work/<scope_id>/room_state.json`（遍历 `iter_work_order_directories()`）。

### 2.3 改写规则

| 原 `status` | 重启后 |
|-------------|--------|
| `processing` | → `stopped`，`stopped_reason=server_restart` |
| `human_intervention` | → `stopped`，`stopped_reason=server_restart` |
| **`failed`** | **不改**（仍为失败，可点重处理） |
| `completed` | 不改 |
| 已是 `stopped` | 不改（可刷新 `stopped_at` 可选，当前实现不刷新） |

### 2.4 与内存任务

重启后 `_running_tasks` 为空；本步骤只纠正 **磁盘** `room_state`，避免前端仍显示「进行中 / 待介入」却无法续跑旧 Agent 会话。

### 2.5 前端感知

列表/详情/live 的 `status === 'stopped'` → 顶栏 **「已停止」**，无需 `boot_id` 比对。

---

## 3. 用户终止：左栏当前节点「停止」

### 3.1 展示条件（同时满足）

- 议题卡片为流水线 **当前节点**（`node.id === room.currentNode`）
- `room.status === 'processing'`
- live **`run_in_progress === true`**
- 非 `human_intervention`（由 `processing` 已隐含）

### 3.2 交互

- 左栏 SOP 议题卡片 **右下角** 停止图标
- 确认后调用 `POST /api/dev/meeting-rooms/{room_id}/stop`

### 3.3 API 行为（本期）

1. `cancel_room_run(room_id)`
2. `room_state.status = stopped`，`stopped_reason = user_stop`
3. `append_history_event`（`room_stopped`）
4. 返回最新 `detail`

### 3.4 TODO-1（本期不实现）

**终止当前智能体处理的完整链路**，后续补全，可能包括：

- 取消 host/worker 的 asyncio Task 与 `agent_pool` 中 `rd_meeting:{room_id}:*` 会话
- 子 Agent 委派中途退出与 trace dump / `reset_agent_node_context`

本期 stop 仅：**取消会议室后台任务 + 标 `stopped`**。

## 4. 重新处理（放宽条件）

### 4.1 前端 `canReprocess`

```text
room 存在 && selectedNode 存在
&& room.status ∈ { failed, human_intervention, stopped }
```

**已移除：**

- 选中节点必须等于当前节点
- `status !== completed`
- `!reprocessing`
- `processing && !runInProgress` 等组合

点击时向 API 传递 **`selectedNode.id`**。

### 4.2 后端 `POST .../reprocess`

请求体：`{ "node_id": "<选中节点>" }`（可选，缺省为当前流水线节点）。

| 分支 | 条件 | 行为 |
|------|------|------|
| **当前节点重处理** | `node_id === room_state.current_node_id` | 见 §4.3 |
| **历史节点重处理** | `node_id !== current_node_id` | 见 §4.4（同阶段内回退光标 + 清理区间后重跑） |

### 4.3 当前节点重处理（对标现有实现）

与现 `reprocess_current_node` 一致：

1. `cancel_room_run(room_id)`
2. Pipeline → `STEP_REPROCESS_PREP`（`_step_reprocess_prep`）
3. 删除 `work/<scope>/agents/<current_node_id>/` 过程目录
4. 删除 `work/<scope>/archive/<当前阶段名>/<current_node_id>/` 归档产出
5. 删除 `host_prompt_snapshot.md`、`hitl.flag.json`；清理 `host_prompt_cache`、`node_metrics[<node>]`、`participants`、`phase`（→ `running`）、`context.host_prompt` 等本节点/门控残留
6. 清除 `hitl_*`、`pending_delivery`、`intervention_kind`、`agents_active`、`stopped_*` 等；`status → processing`
7. 流水线：`STEP_NODE_INIT` → 组装主控提示词 → `schedule_run_node`

**不是**全量重开会议室。

从 `failed` / `human_intervention` / `stopped` 进入时，prep 后同样回到 `processing` 并调度。

### 4.4 历史节点重处理（已实现）

**同阶段内回退光标 + 清理区间后重跑**，代码锚点：`service.py::_reprocess_historical_node`。

1. `_validate_historical_reprocess_target`：校验 target 在 current 之前、同阶段、非 system 节点；返回 `[target..current]` 闭区间节点 id 列表
2. `cancel_room_run(room_id)`
3. `dev.status.current_node_id = target`，`stage_id` / `sop_node_display` / `local_process_state` 同步更新
4. `sync_room_state_from_dev`：room_state 光标回退
5. `patch_userwork_summary`：工单侧 `sop_node` 同步
6. Pipeline `STEP_REPROCESS_PREP`：`context.reprocess_node_ids = [target..current]`，`context.reprocess_historical_target = target`
7. 清理区间内各节点的 `agents/` 过程目录、`archive/` 归档、`host_prompt_snapshot.md`、`hitl.flag.json` 等
8. 流水线：`STEP_NODE_INIT` → 组装主控提示词 → `schedule_run_node`

**跨阶段重处理**：返回 `400 cross_stage_reprocess_forbidden`（不允许跨阶段回退）。
**system 节点重处理**：返回 `400 system_node_reprocess_forbidden`。

## 5. 状态 × 操作矩阵

| room.status | 左栏「停止」 | 「重新处理」（任意选中节点） |
|-------------|-------------|------------------------------|
| `processing` + run | ✅ 仅当前节点 | ❌ |
| `human_intervention` | ❌ | ✅（当前节点→§4.3；历史→§4.4） |
| `failed` | ❌ | ✅ |
| `stopped` | ❌ | ✅ |
| `completed` | ❌ | ❌ |
