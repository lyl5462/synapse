# 研发会议室测试手册

面向 `req_clarify` 及后续 SOP 节点的手工与自动化验收。运行前请确保：

- Python 3.11+ 虚拟环境已安装：`pip install -e ".[dev]"`
- 后端：`synapse serve`（默认 API 端口见配置，常见为 16185）
- 前端（测 UI 时）：`cd apps/setup-center && npm run tauri dev` 或 Vite dev
- Host / Worker Profile 已在 Agents 中配置业务技能（需求澄清等）；会议室 SKILL 由系统注入，无需在节点配置里填 `skill_ids`

## 1. 自动化单测

```bash
pytest tests/unit/test_rd_meeting_phase5_node_enable.py -q
pytest tests/unit/test_rd_meeting_user_context_intervene.py -q
pytest tests/unit/test_rd_meeting_live_phase_artifacts.py -q
pytest tests/unit/test_rd_meeting*.py -q
```

## 2. Dry-run（不调用 LLM）

设置环境变量后启动服务：

```bash
set SYNAPSE_MEETING_ROOM_DRY_RUN=1
synapse serve
```

Dry-run 模式下，`open_meeting` 和 `reprocess` 均不会实际调用 LLM，而是模拟智能体按 NODE_OUTPUTS 约定落盘。

## 3. P0：开会与人工续跑

| 步骤 | 操作 | 期望 |
|------|------|------|
| 1 | `POST /api/dev/meeting-rooms/open` | 主控 prompt 组装后自动后台执行当前节点；`room_state.status=processing` |
| 2 | 会中填写 HITL 问卷并提交，`message_type=instruction` | `user_context_pending` 注入后 `resume_run`；节点继续 |
| 3 | 运行中发送普通聊天，`message_type=chat` | 仅写入 history / pending，**不**触发 `schedule_run_node` |
| 4 | 结果确认门控提交 approve | 校验智能体已落盘的 NODE_OUTPUTS 约定 `.md`，推进下一节点 |

## 4. P1：Live 与运行时上下文

| 步骤 | 操作 | 期望 |
|------|------|------|
| 1 | 打开会议室弹窗 | 每 3s 请求 `GET /api/dev/meeting-rooms/{id}/live` |
| 2 | Host 执行 `delegate_to_agent` | history 出现 `delegation_started` / `delegation_finished`；live 含 `sub_agents` |
| 3 | 查看节点 prompt（dry-run 日志） | 含 `WORK_ORDER_DIR`、`GITNEXUS_URL`、`DEMAND_DESC` 等自动注入段 |

## 5. P2：配置与产物

| 步骤 | 操作 | 期望 |
|------|------|------|
| 1 | 会议室配置抽屉保存节点覆盖 | **无** `skill_ids` 字段；PUT 不持久化该键 |
| 2 | 结果确认通过 | `work/<scope>/archive/<stage>/<node>/` 含 NODE_OUTPUTS 约定名（如 `需求澄清.md`），无流程代写的 `result.md` |
| 3 | `room_state.phase` | 随门控在 `running` / `clarify_gate` / `result_gate` / `exception_gate` / `completed` 间变化 |

## 6. P3：异常与校验

| 步骤 | 操作 | 期望 |
|------|------|------|
| 1 | 模拟委派失败（错误 agent_id 或断网） | 异常门控（无系统内置题目，可对话区说明或主控输出 `hitl-questionnaire`）；`intervention_kind=exception`；`phase=exception_gate` |
| 2 | 确认通过但产物过短或无 `#` 标题 | `confirm_node_delivery` 返回校验错误 |
| 3 | 确认通过但缺少 NODE_OUTPUTS 约定文件 | `node_archive_validation_failed` |

## 7. P4：停止与重处理

| 步骤 | 操作 | 期望 |
|------|------|------|
| 1 | 节点运行中点击「停止」 | `POST .../stop` → `room_state.status=stopped`；`stopped_reason=user_stop` |
| 2 | stopped 状态下点击「重新处理」当前节点 | `POST .../reprocess` → 清理过程数据 → `status=processing` → 从 node_init 重跑 |
| 3 | stopped 状态下点击「重新处理」同阶段历史节点 | `POST .../reprocess { node_id }` → 光标回退 → 清理 [target..current] 区间 → 从 target 重跑 |
| 4 | 尝试重处理跨阶段历史节点 | 返回 `400 cross_stage_reprocess_forbidden` |
| 5 | 尝试重处理 system 节点 | 返回 `400 system_node_reprocess_forbidden` |
| 6 | 服务重启后 | 所有 `processing`/`human_intervention` 的会议室标为 `stopped`，`stopped_reason=server_restart` |
